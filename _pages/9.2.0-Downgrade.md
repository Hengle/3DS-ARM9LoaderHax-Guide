有些区域的机器会被降级到9.1.0或9.0.0等版本。这和降级至9.2.0一样都能达到我们的目的。
{: .notice--success}

如果你遇到了这个错误并且无法使用恢复模式，只要你的3DS完全丢失了与任何接入点的连接（如接入点超出了范围），你就可以修改连接设置。
{: .notice--info}

目前版本的sysDowngrader（适用于9.2.0以上版本的系统）使用了一个不稳定的漏洞，所以可能要试很多次才能成功。
{: .notice--info}

{% capture notice-3 %}
如果PlaiSysUpdater在降级过程中奔溃了（这时不时会发生），你可以通过长按电源键关机。你的系统将会是“部分降级的”，但这可以恢复。

尽管一个部分降级的系统可能会在系统版本里面显示9.2.0，但这是不够的。你必须从第二部分开始再次降级，以便重新安装之前没有成功安装的文件。
{% endcapture %}

<div class="notice--info">{{ notice-3 | markdownify }}</div>

如果你的系统版本在11.0.0至11.2.0之间，你必须已经进行了[NFIRM降级](nfirm-downgrade)才能顺利完成本节操作。
{: .notice--warning}

**你必须使用对应于你设备/区域的降级包，否则会变砖。**
{: .notice--danger}

#### 操作指南

##### 章节 I - 准备工作

3. 复制sysDowngrader压缩包的`3ds`文件夹到你SD卡的根目录
4. 复制nimSM压缩包的`3ds`文件夹到你SD卡的根目录
4. 删除任何在SD卡上的`updates`文件夹，如果有的话
5. 从固件降级包中拷贝`updates`文件夹到你的SD卡根目录
6. 将SD卡插回你的3DS

##### 章节 II - 降级

1. 进入自制程序启动器
2. 运行nimSM
3. 按(Y)键备份并清零nim存档
  + 这将防止降级系统的时候造成"软变砖"
  + "软变砖"是指系统启动时显示错误画面，*只能* 使用恢复模式（A + R + L + 十字上键）升级到最新版本的系统来修复。
4. 等待nim存档备份和清零成功
  + 如果程序在尝试进行svchax的时候卡住，长按电源键关机并重试(可能要试很多次)
  + 如果程序在尝试终止fs module时卡住，长按电源键关机并重试(可能要试很多次)
5. 按(开始)键重启
2. 打开sysDowngrader
5. 按(Y)降级到9.2.0
7. 等待，即使看起来卡住了（屏幕显示完全不变）也不要触碰任何东西
8. 如果卡住差不多有30秒，可以关机再试一次
9. 如果在安装开始前出现错误信息，长按电源键关机，然后再试一次（可能要试很多次）
10.  如果你在安装中途出现错误，参见本部分一开始对“部分降级”的说明。
11. 如果你卡在"Rebooting in 10 seconds"这一行超过10秒钟，可以通过长按电源键关机
13. 进入SD卡的`/Nintendo 3DS/(32 字符 ID)/(32 字符 ID)/extdata/00000000/`目录以清除主界面菜单的extdata
    + 欧版: 删除 `00000098` 
    + 日版: 删除 `00000082`
    + 美版: 删除 `0000008f`
    + 韩版: 删除 `000000A9`
    + 台版: 删除 `000000B1`
14. 将SD卡插回你的3DS，并启动机器
12. 如果在降级后出现黑屏，参见[问题排查](troubleshooting#ts_sys_down)
